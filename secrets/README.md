# Credential drop-zone

`secrets/local/` is the place for real Docket credentials. Its contents are
ignored by Git except for its README. Never paste secrets into source files,
issues, test fixtures, or chat transcripts.

## Prepare the directory

```bash
mkdir -p secrets/local
chmod 700 secrets/local
```

Place or generate these files in `secrets/local/`:

| File | Used by | Needed |
|---|---|---|
| `discord_bot_token` | Hermes | Milestone 0 live Discord smoke |
| `docket_to_hermes_token` | Docket MCP authentication | Before Hermes connects |
| `hermes_to_docket_token` | Trusted plugin callback API | Before plugin smoke |
| `interaction_signing_key` | Future signed Discord controls | Milestone 2 |
| `postgres_password` | Local PostgreSQL role | Generated by Docket production setup |
| `searxng_secret` | Private SearXNG session signing | Generated by Docket production setup |
| `google_oauth_client.json` | Docket Google adapter | Milestone 2 |
| `google_oauth_token.json` | Docket Google adapter | Generated by Docket; do not create manually |

Generate service secrets locally:

```bash
openssl rand -hex 32 > secrets/local/docket_to_hermes_token
openssl rand -hex 32 > secrets/local/hermes_to_docket_token
openssl rand -hex 32 > secrets/local/interaction_signing_key
chmod 600 secrets/local/*
```

Generate and install the PostgreSQL and SearXNG credentials without displaying
them:

```bash
uv run docket-production-config
```

SearXNG receives its credential through a Compose secret; it is not copied into
`.env` or passed on its command line.

Do not copy `secrets/smoke/google_oauth_token.json` into this directory. After
placing the Google OAuth client JSON, validate it and let Docket create the token:

```bash
uv run docket-google-auth status --credentials-dir secrets/local
scripts/setup-google-oauth.sh
```

The client must be downloaded as a Google OAuth **Desktop app** credential. The
setup uses a localhost callback, requests offline access, requires a refresh
token, strips the short-lived access token before persistence, and writes the
result atomically with file mode `0600`. The recognized smoke placeholder is
replaced automatically; any other existing token requires `--force`.
The command refuses to write into the checked-in `secrets/smoke/` directory.

The default `workspace` profile is the operator-approved bundle:

```text
https://www.googleapis.com/auth/calendar.events
https://www.googleapis.com/auth/documents
https://www.googleapis.com/auth/gmail.modify
https://www.googleapis.com/auth/spreadsheets
```

To intentionally request a narrower token instead, name every desired profile:

```bash
scripts/setup-google-oauth.sh \
  --scope-profile calendar \
  --scope-profile gmail-read
```

Available profiles are `workspace`, `calendar`, `docs`, `sheets`, `gmail-read`,
and `gmail-modify`. The `gmail-modify` permission is broader than Docket's
planned archive/mark-read actions, so Docket continues to prohibit send/reply
endpoints in its registry and adapters. Sheets and Docs authorization likewise
does not expose tools before Docket implements and gates those adapters. For a
remote host, use `--no-browser --port PORT` and arrange an SSH tunnel so your
browser can reach that localhost callback.

Set `DOCKET_CREDENTIALS_DIR=./secrets/local` in `.env` only after replacing the
dummy service values. Keep `DOCKET_EXTERNAL_CALLS_ENABLED=false` until the applicable
milestone's fake and adversarial tests pass.

Run `scripts/prepare-hermes-home.sh` to copy non-secret configuration and the
Discord/MCP tokens into the ignored Hermes runtime directory.
